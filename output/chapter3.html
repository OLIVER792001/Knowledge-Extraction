This page documents my portion of the WishGraph knowledge representation project. The workflow moves through four sequential steps: establishing the theoretical framework and competency questions (Step 1), designing the T-Box ontology skeleton (Step 2), using multimodal AI prompting to extract A-Box instance data from collected meme images (Step 3), and finally querying the resulting graph with SPARQL to surface humanistic insights (Step 4).

The 5P Theoretical Framework

Affective (anxiety relief, hope) & Meme (virality, community bonding)

This project adapts Rhodes' 4P creativity model (Person, Process, Product, Press) to the domain of cyber-wishing, adding Platform as a fifth dimension to capture how platform affordances (comments, reposts, hashtags, recommendation algorithms) shape the form, circulation and public reach of digital rituals. Theoretical grounding draws on Digital Folklore (Howard 2008), Ritual Theory (Bell 1992; Turner 1969), Meme Theory (Shifman 2014; Milner 2016) and Affective Practice (Ahmed 2004).

📌 CQ Alignment Note — Cross-Platform Symbol Distribution (CQ6 Addition)

The existing 5 CQs already cover the 4P framework well. To deepen Step 4's analytical power, a sixth competency question has been added:

CQ6 — Platform–Symbol Co-distribution

How does the distribution of specific ritual symbols (e.g., "Koi" vs. "Offer Bodhisattva") differ across platforms (e.g., RedNote vs. LinkedIn)? This demonstrates the value of the circulatesOnPlatform property and adds "digital humanities" analytical depth to the knowledge graph.

3.2 — AI Prompt Engineering: Meme 4 Case Study

The extraction prompt is not a simple "what is in this image?" question. Instead it follows a three-layer reasoning logic that progressively deepens from visual observation to ontological abstraction:

🔍 Prompt 1 — Identification → 🔗 Prompt 2 — Relational Mapping → 🧠 Prompt 3 — Logical Abstraction

By answering just 4–5 targeted meme images with this three-question protocol, we extract high-quality A-Box instance data that directly populates the WishGraph ontology. Below is the full demonstration using Meme 4 (Health-Anxious User).

1 Prompt 1 🔍 Identification — Who, What, Where?
🖼 Meme Image
📝 Prompt 1
Task
Please analyze this screenshot (including the meme image and its social media context).
① Identify the subject (Person): Based on the posting context, profile picture, or comment content, infer the social identity of the participants (e.g., Gamer, SingleYoungAdult, Researcher, etc.).
② Pinpoint the pressure points (Pressure): What anxieties, desires, or uncertainties are they expressing?
③ Identify the Cyber Product: What is the central visual object in the image? (e.g., Love Fortune Wallpaper, Debugging Bodhisattva, etc.)
④ Identify the Platform: Based on UI features, determine which platform this occurred on (e.g., RedNote, Bilibili, LinkedIn, etc.).
🤖 AI Output
2 Prompt 2 🔗 Relational Mapping — Cultural & Behavioral Logic
🖼 Meme Image
📝 Prompt 2
Task
Based on the previous analysis, please further explore the underlying traditions and behavioral logic.
① Trace the Cultural Target (CulturalTarget): What traditional folk symbols, deities, or religious archetypes does this cyber-amulet correspond to? (e.g., Yuelao, Caishen, Koi culture)
② Analyze Digital Practices (DigitalPractice): What actions do users take to fulfill this "wish"? (e.g., sharing, commenting "99", placing the item next to physical devices, etc.)
③ Extract Ritual Symbols (RitualSymbol): Which specific visual elements in the image convey a heightened sense of "ritual" and "sacredness"? (e.g., peach blossom background, gold coin emoji, specific "99" comment text, ASCII art)
Please analyze how these elements work together to simulate a traditional ritual.
🤖 AI Output
3 Prompt 3 🧠 Logical Abstraction — Knowledge Engineer Summary
🖼 Meme Image
📝 Prompt 3
Task
Finally, please provide a high-level summary from the perspective of a "knowledge engineer" so that I can convert it into JSON/RDF format.
Summary of TransformationMechanism: How are traditional symbols transformed into "cyber" forms? Is it through interface simulation, homophonic transformation, or aestheticization?
Define AffectiveFunction: What psychological good does this behavior provide to users? (e.g., communal encouragement, emotional reassurance, anxiety relief)
Define MemeFunction: How does it spread on social networks? (e.g., comment ritual, fandom joke, participatory blessing)
Final Task: Please integrate all findings and output them strictly in the following JSON format:
{"person":"…","pressure":"…","cyberProduct":"…","culturalTarget":"…","digitalPractice":"…","platform":[…],"transformationMechanism":[…],"affectiveFunction":[…],"memeFunction":[…],"ritualSymbol":[…]}

4 Step 4: SPARQL Insights Data Queries & Humanistic Analysis

CQ1 — Actors and Pressures

Which types of users engage in cyber-wishing, and what pressures or desires motivate them?

# CQ1: Who wishes, and why? SELECT ?case ?personLabel ?pressureLabel WHERE { ?case a wg:CyberWishingCase . ?case wg:hasPerson ?person . ?case wg:hasPressure ?pressure . ?person rdfs:label ?personLabel . ?pressure rdfs:label ?pressureLabel . } ORDER BY ?personLabel

CQ2 — Traditional References → Cyber Products

How are traditional deities transformed into cyber products and meme-based wishing devices?

SELECT ?case ?cyberProductLabel ?culturalTargetLabel ?mechanismLabel WHERE { ?case a wg:CyberWishingCase . ?case wg:usesCyberProduct ?product . ?product rdfs:label ?cyberProductLabel . OPTIONAL { ?case wg:alignsWithCulturalTarget ?target . ?target rdfs:label ?culturalTargetLabel . } OPTIONAL { ?case wg:hasTransformationMechanism ?mech . ?mech rdfs:label ?mechanismLabel . } }

CQ3 — Digital Practices and Platforms

What digital practices mediate cyber-wishing across different platforms?

SELECT ?platformLabel (GROUP_CONCAT(DISTINCT ?practiceLabel ; separator=" | ") AS ?practices) WHERE { ?case wg:circulatesOnPlatform ?platform . ?case wg:performsPractice ?practice . ?platform rdfs:label ?platformLabel . ?practice rdfs:label ?practiceLabel . } GROUP BY ?platformLabel

CQ4 — Affective Functions

What affective functions do cyber-wishing practices serve for different user groups?

SELECT ?personLabel (GROUP_CONCAT(DISTINCT ?affLabel ; separator=" | ") AS ?affFunctions) WHERE { ?case wg:hasPerson ?person . ?case wg:servesAffectiveFunction ?af . ?person rdfs:label ?personLabel . ?af rdfs:label ?affLabel . } GROUP BY ?personLabel

CQ5 — Meme Functions and Ritual Symbols

How do meme functions and ritual symbols support the circulation of cyber-wishing practices?

SELECT ?case ?memeFunctionLabel ?ritualSymbolLabel ?platformLabel WHERE { ?case a wg:CyberWishingCase . OPTIONAL { ?case wg:servesMemeFunction ?mf . ?mf rdfs:label ?memeFunctionLabel . } OPTIONAL { ?case wg:usesRitualSymbol ?rs . ?rs rdfs:label ?ritualSymbolLabel . } OPTIONAL { ?case wg:circulatesOnPlatform ?pl . ?pl rdfs:label ?platformLabel . } } ORDER BY ?case

CQ6 — Platform–Symbol Co-distribution

How does the distribution of specific ritual symbols (e.g., "Koi" vs "Offer Bodhisattva") differ across platforms (e.g., RedNote vs LinkedIn)?

# CQ6: Platform-symbol co-distribution — which ritual symbols appear on which platforms? SELECT ?symbolLabel ?platformLabel (COUNT(DISTINCT ?case) AS ?caseCount) WHERE { ?case a wg:CyberWishingCase . ?case wg:usesRitualSymbol ?symbol . ?case wg:circulatesOnPlatform ?platform . ?symbol rdfs:label ?symbolLabel . ?platform rdfs:label ?platformLabel . } GROUP BY ?symbolLabel ?platformLabel ORDER BY DESC(?caseCount) ?platformLabel

Humanistic insight: This query reveals whether platform affordances select for particular ritual symbol vocabularies — e.g., koi imagery concentrating on lifestyle platforms (RedNote) while career deities (Offer Bodhisattva) dominate professional networks (LinkedIn, Maimai). This validates the circulatesOnPlatform property's analytical power.
